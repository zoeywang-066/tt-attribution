# TT 数据更新报告规则

本项目自 2026-07-17 起只维护单周报：

- 单周报：`index.html`，每周一更新上上周日到上周六，对比前一个 7 天。
- 双周报：`biweekly.html` 停止更新，仅保留现有历史快照。

## 触发逻辑

单周报使用以下触发逻辑：

- 统计粒度：国家 / OS / app id。
- 指标：BI CPI 或 DNU。
- 触发条件：BI CPI 或 DNU 在统计周期内相对对比周期波动超过 10%，上涨和下降都触发。
- Benchmark：优先使用 `ads-bpd-guard-sandbox.sherry` 沙盒表作为波动和大盘 benchmark。
- 明细归因：使用 Moloco BQ / `ads-bpd-guard-china.athena` 明细表拆解 campaign、投放事件、媒体、creative、bundle。

## 归因输出要求

不要只罗列数字。每个触发国家必须给一条完整、可直接复制给客户的一句话总结：

> 这个国家在本周期 BI CPI / DNU 波动的主要原因是：某个投放事件或 campaign 成本上升 / 降量，或某些 top 媒体 CPM/CPI 上涨，或某个创意组/版位衰败，或 bundle 子渠道结构变化。

固定句式按以下顺序生成：

> 本周 DNU[上涨/下降]主要受[消耗变化/CPI变化]影响；周期内 CPI[上涨/下降]主要归因于[内部竞争加剧/事件结构变化/媒体成本变化/创意效率变化]；细分数据如下：[事件]事件消耗占比 X%、CPI ±X%；媒体侧[媒体名称]消耗占比 X%、CPI ±X%、CPM ±X%。大盘对比：非游戏大盘 CPI ±X%，客户相对大盘 ±Xpp。后续优化动作：[具体动作]。

示例：

> 本周 DNU 下降主要受 CPI 上涨影响；周期内 CPI 上涨主要归因于内部竞争加剧；细分数据如下：install 事件消耗占比 56.2%、CPI +17.9%；媒体侧 com.miui.videoplayer 消耗占比 3.3%、CPI +50.0%、CPM +37.9%。大盘对比：非游戏大盘 CPI +6.0%，客户涨幅高于大盘 5.0pp。后续优化动作：下调高成本 install 事件预算，屏蔽 com.miui.videoplayer 等高 CPI 媒体，并移除 CPI 高于目标且 IPM 持续下降的创意。

“内部竞争加剧”不能仅凭 CPI 上涨推断；只有客户 CPM 明显上涨、涨幅高于非游戏大盘，或多个主要媒体 CPM 同步上涨时才可使用。否则应写成已被数据支持的直接原因，例如“install 事件成本上涨”或“高成本媒体占比提升”。

结论下面再列数据证据：

- BI CPI / DNU 的变化幅度。
- 国家预算/消耗变化：DNU 波动必须解释预算是否调整、消耗是否上升/下降，以及 CPI 成本变化是否导致同预算获量减少或增加。
- 主要 campaign / 投放事件的消耗占比、CPI、变化。
- top 媒体的占比变化、CPM、CPI、IPM。
- top creative / creative format 的 CPI、IPM、占比变化。
- top bundle id / publisher 的占比变化、CPI、CPM、IPM。
- 非游戏行业大盘 CPM / CPI benchmark 对比。
- 操作历史：优先读取 campaign / ad group / creative 的 edit history，记录操作时间、操作对象、前值和后值。
- 预算调整：有 edit history 时明确写预算从 A 调到 B；无日志时只能写国家/ campaign 消耗变化为“放量或收量信号”，不能直接等同于预算修改。
- 媒体屏蔽、创意移除和 campaign 暂停：若上期有显著消耗、本期归零或大幅降量，先标记为“疑似操作影响”；只有匹配到 edit history 后才标记“已确认”。
- 操作效果：高成本媒体/创意退出且整体 CPI 下降，可归因为操作改善；低成本流量退出且整体 CPI 上涨，需要提示该操作可能带来结构性成本上升。

## 优化建议输出要求

每个触发国家要给可执行动作，优先按以下顺序：

- 降低高成本事件或高成本 campaign 的预算。
- 如果 DNU 下滑来自国家预算下降，需要明确是主动预算调整还是成本上升导致消耗转移；如果 DNU 下滑来自 CPI 上涨，需要优先压低高成本流量。
- 屏蔽或降量高成本媒体 / exchange / bundle。
- 移除高 CPI、IPM 衰退的创意组。
- 补充测试新创意，特别是当前主力版位疲劳时。
- 若大盘也同步上涨，沟通口径要标注“行业大盘上行”，避免把全部原因归到投放操作。

一句话中的动作必须与前文异常对象一一对应：

- 高成本事件主导：写明事件名称，并建议下调该事件预算或向低成本事件迁移预算。
- 高 CPI / CPM 媒体主导：写明媒体或 bundle 名称，并建议屏蔽或降量。
- 创意 IPM / CVR 衰退主导：建议移除具体高成本创意，并补充新创意测试。
- 消耗增长导致 DNU 上涨且 CPI 稳定：保留扩量，不给降预算建议。
- 大盘同步上涨且客户涨幅接近大盘：以控制高成本流量、稳定预算为主，不把行业上涨全部归因于内部操作。

## 归因证据等级

- 已确认：edit history 有明确时间戳、对象和前后值，且操作后指标方向一致。
- 高可能：投放明细显示对象在操作窗口后归零/显著降量，且其历史成本与整体变化方向吻合，但没有读取到编辑日志。
- 相关但未证实：预算/消耗、媒体或创意结构与 CPI 同期变化，仅可作为解释线索，不能表述为确定因果。

## 页面结构

- 单周报保留当前客户周报页。
- 历史双周报页面不再刷新或调整。
- 每个归因详情块必须先展示“结论 + 主要原因判断”，然后才展示明细表。
